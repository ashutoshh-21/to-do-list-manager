# 📝 ToDo List Manager — CLI Application

A complete command-line task management system built in Python with SQLite storage.  
Manage your daily tasks with priorities, categories, due dates, search, filtering, and statistics — all from your terminal.

---

## 🚀 Features

- ✅ Add, view, update, delete, and complete tasks
- 🔴🟡🟢 Priority levels: High, Medium, Low
- 🗂️ Custom categories
- 📅 Due dates with overdue detection
- 🔍 Search by keyword
- 📊 Progress statistics with visual progress bar
- 📤📥 Export / Import tasks as JSON
- 💾 Persistent storage via SQLite
- 🖥️ Interactive menu + direct CLI commands

---

## 🛠️ Requirements

- Python **3.7+**
- No external libraries required — uses Python standard library only (`sqlite3`, `json`, `datetime`, `sys`, `os`)

---

## ⚙️ Setup & Installation

### 1. Clone the repository

```bash
git clone https://github.com/<your-username>/todo-list-manager.git
cd todo-list-manager
```

### 2. (Optional) Create a virtual environment

```bash
python -m venv venv
source venv/bin/activate        # Linux / macOS
venv\Scripts\activate           # Windows
```

### 3. Run the application

```bash
python todo.py
```

> On first run, a `tasks.db` SQLite database file is automatically created in the project directory.

---

### Direct CLI Commands

You can also run commands directly without entering the menu:

| Command | Description | Example |
|---------|-------------|---------|
| `python todo.py` | Launch interactive menu | — |
| `python todo.py add "Title"` | Add a task | `python todo.py add "Buy milk"` |
| `python todo.py list` | List pending tasks | — |
| `python todo.py done <id>` | Mark task complete | `python todo.py done 3` |
| `python todo.py delete <id>` | Delete a task | `python todo.py delete 5` |
| `python todo.py stats` | Show statistics | — |
| `python todo.py help` | Show help | — |

#### Options for `add` command:

```bash
python todo.py add "Task Title" --priority high --category Work --due 2025-06-15 --desc "Optional description"
```

| Flag | Values | Default |
|------|--------|---------|
| `--priority` | `high`, `medium`, `low` | `medium` |
| `--category` | Any string | `General` |
| `--due` | `YYYY-MM-DD` | None |
| `--desc` | Any string | Empty |

## 🗄️ Data Storage

All tasks are stored in a local **SQLite** database (`tasks.db`).  
The schema includes: `id`, `title`, `description`, `priority`, `category`, `due_date`, `completed`, `created_at`, `completed_at`.

You can also export to JSON (`tasks_export.json`) using option 12 in the menu.

## 🧪 Testing

The application handles the following edge cases:
- Empty task title → error message
- Invalid priority input → error message
- Invalid date format → error message
- Non-existent task ID → error message
- Deleting a task requires confirmation
